Fig. 1 (Rev.). The execution process of SRSPSQL with integrated GAT.
Stage ① rewrites ambiguous questions using LLM + database content (BERT table selection; GloVe row filtering, top-k=5).
Stage ② builds a Schema Graph, applies a Graph Attention Network (GAT) over table/column nodes with FK edges to produce schema-aware embeddings, then uses GAT-informed pruning to construct the full-schema prompt for initial SQL (Pre_SQL).
Stage ③ parses Pre_SQL via normalization → skeleton + schema extraction, filters examples by AST similarity, applies dynamic schema pruning (↓44% token overhead), and generates the final Fin_SQL.
